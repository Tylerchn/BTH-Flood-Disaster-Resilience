# 从零开始：将博士论文可视化仪表盘部署为网站

## 完整手把手教程（适用于零基础）

---

## 总览：你需要做什么

整个过程分为 5 步，预计耗时 30-60 分钟：

```
第1步 安装必要工具（10分钟）
  ↓
第2步 在 GitHub 上创建仓库（5分钟）
  ↓
第3步 将项目文件上传到 GitHub（10分钟）
  ↓
第4步 开启 GitHub Pages 自动部署（5分钟）
  ↓
第5步 网站上线！访问 https://你的用户名.github.io/仓库名/
```

后续如果要替换为真实数据，再看"第6步"。

---

## 第1步：安装必要工具

### 1.1 安装 Node.js

Node.js 是运行网站开发工具的基础环境。

1. 打开浏览器，访问 https://nodejs.org/
2. 点击左边的 **LTS（长期支持版）** 按钮下载
3. 双击下载的安装包，一路点"下一步"即可
4. 安装完成后，打开 **命令提示符**（Windows键 → 输入 `cmd` → 回车）
5. 输入以下命令验证安装成功：

```bash
node --version
npm --version
```

看到版本号（比如 `v20.11.0` 和 `10.2.0`）就说明安装成功了。

### 1.2 安装 Git

Git 是将代码上传到 GitHub 的工具。

1. 访问 https://git-scm.com/download/win
2. 下载并安装（安装过程中所有选项保持默认即可）
3. 安装完成后，在命令提示符中验证：

```bash
git --version
```

看到 `git version 2.xx.x` 就说明安装成功了。

### 1.3 配置 Git 身份信息

在命令提示符中执行（把引号里的内容换成你自己的）：

```bash
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"
```

## 第2步：在 GitHub 上创建仓库

1. 打开浏览器，登录 https://github.com
2. 点击右上角的 **+** 号 → **New repository**
3. 填写仓库信息：
   - **Repository name**: `bth-resilience-dashboard`（或者你喜欢的英文名字）
   - **Description**: `京津冀城市群洪涝韧性全过程可视化仪表盘`
   - 选择 **Public**（公开，免费用户只能用公开仓库部署网站）
   - **不要**勾选 "Add a README file"（我们会自己上传）
4. 点击 **Create repository**

创建完成后，你会看到一个页面显示仓库地址，类似：
```
https://github.com/你的用户名/bth-resilience-dashboard.git
```
**复制这个地址**，后面要用。

## 第3步：将项目文件上传到 GitHub

### 3.1 解压项目文件

把从 Claude 下载的 `bth-resilience-dashboard.zip` 解压到你电脑上一个方便找到的位置，比如 `D:\Projects\bth-resilience-dashboard\`。

解压后的文件结构应该是：

```
bth-resilience-dashboard/
├── .github/
│   └── workflows/
│       └── deploy.yml          ← 自动部署配置（不用改）
├── public/                     ← 静态资源（后续放数据JSON）
├── scripts/
│   └── prepare_data.py         ← 数据准备脚本（后续用）
├── src/
│   ├── main.jsx                ← 入口文件（不用改）
│   └── Dashboard.jsx           ← 仪表盘主组件
├── .gitignore
├── index.html
├── package.json
├── vite.config.js              ← ⚠️ 可能需要改仓库名
└── 部署教程.md                  ← 本文件
```

### 3.2 ⚠️ 修改仓库名配置

用记事本（或 VS Code）打开 `vite.config.js`，检查第 8 行：

```javascript
base: '/bth-resilience-dashboard/',
```

**如果你的 GitHub 仓库名不是 `bth-resilience-dashboard`**，把这里改成你的实际仓库名。比如你的仓库叫 `my-dashboard`，就改成：

```javascript
base: '/my-dashboard/',
```

### 3.3 在本地安装依赖并测试

打开命令提示符，依次执行：

```bash
# 进入项目文件夹（改成你的实际路径）
cd D:\Projects\bth-resilience-dashboard

# 安装项目依赖（会自动下载 React 等库，需要等1-2分钟）
npm install

# 启动本地预览服务器
npm run dev
```

你会看到类似这样的输出：

```
  VITE v6.0.0  ready in 300 ms

  ➜  Local:   http://localhost:5173/bth-resilience-dashboard/
  ➜  press h + enter to show help
```

**在浏览器中打开这个网址**，你应该能看到仪表盘在本地正常运行了！

确认没问题后，按 `Ctrl+C` 停止服务器。

### 3.4 上传到 GitHub

在同一个命令提示符窗口中，依次执行以下命令：

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 创建第一次提交
git commit -m "初始化：京津冀洪涝韧性可视化仪表盘"

# 设置主分支名为 main
git branch -M main

# 关联到你的 GitHub 仓库（⚠️ 替换成你自己的仓库地址）
git remote add origin https://github.com/你的用户名/bth-resilience-dashboard.git

# 推送到 GitHub
git push -u origin main
```

执行 `git push` 时，系统可能会弹出登录窗口，按提示登录你的 GitHub 账号即可。

如果遇到认证问题，参见本文最后的"常见问题"部分。

## 第4步：开启 GitHub Pages

1. 回到浏览器，打开你的 GitHub 仓库页面
2. 点击 **Settings**（设置）标签
3. 在左侧菜单中找到 **Pages**
4. 在 "Build and deployment" 部分：
   - **Source** 选择 **GitHub Actions**（不是 "Deploy from a branch"）
5. 不需要其他操作，保存即可

### 4.1 等待自动部署

1. 回到仓库主页，点击 **Actions** 标签
2. 你会看到一个正在运行的工作流（黄色圆圈表示进行中）
3. 等待 1-3 分钟，变成绿色勾就表示部署成功了

### 4.2 访问你的网站

部署成功后，你的网站就上线了！地址是：

```
https://你的用户名.github.io/bth-resilience-dashboard/
```

比如你的 GitHub 用户名是 `tangxxx`，那网址就是：
```
https://tangxxx.github.io/bth-resilience-dashboard/
```

🎉 **恭喜！你的论文可视化网站已经上线了！**

## 第6步：替换为真实数据

目前仪表盘使用的是模拟数据。要换成你论文的真实数据，需要两步操作。

### 6.1 用 Python 脚本准备数据

1. 打开 `scripts/prepare_data.py`
2. 修改文件开头的路径配置，指向你的实际数据文件：

```python
DATA_ROOT = r"E:\大论文数据处理\26年重新计算"
```

3. 根据你的 CSV 文件的实际列名，修改脚本中标注了 `⚠️` 的地方
4. 运行脚本：

```bash
cd D:\Projects\bth-resilience-dashboard
python scripts/prepare_data.py
```

5. 脚本会在 `public/` 文件夹中生成 `watershed_data.json`

### 6.2 修改 Dashboard.jsx 加载真实数据

打开 `src/Dashboard.jsx`，找到文件开头的模拟数据生成部分（大约前 100 行），
把 `generateWatershedData()` 函数替换为从 JSON 加载数据。

具体做法是在 Dashboard 组件中使用 `fetch` 加载数据：

```jsx
// 在 Dashboard 组件内部，替换原来的 const WATERSHED_DATA = generateWatershedData();

const [watershedData, setWatershedData] = useState([]);
const [loading, setLoading] = useState(true);

useEffect(() => {
  fetch(import.meta.env.BASE_URL + 'watershed_data.json')
    .then(res => res.json())
    .then(data => {
      setWatershedData(data.units);
      setLoading(false);
    })
    .catch(err => {
      console.error('数据加载失败:', err);
      // 降级为模拟数据
      setWatershedData(generateWatershedData());
      setLoading(false);
    });
}, []);

if (loading) return <div>加载数据中...</div>;
```

然后把组件中所有引用 `WATERSHED_DATA` 的地方改为 `watershedData`。

这一步如果你觉得复杂，可以直接把改好的代码发给我看，我帮你调整。

### 6.3 添加真实地图（可选进阶）

如果想用真实的汇水单元矢量边界而不是圆点，需要：

1. 在 `prepare_data.py` 中取消 `export_geojson()` 的注释
2. 运行脚本生成 `public/watersheds.geojson`
3. 在 Dashboard 中引入地图库（如 Leaflet 或 Mapbox GL JS）
4. 用 GeoJSON 渲染真实的汇水单元多边形

这一步比较复杂，建议先用当前的圆点版本上线，后续我们可以一起迭代升级。

## 常见问题

### Q: git push 时提示认证失败怎么办？

GitHub 从 2021 年起不再支持密码登录。你需要创建一个 Personal Access Token：

1. 登录 GitHub → 点右上角头像 → **Settings**
2. 左侧最底部 → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
3. 点 **Generate new token (classic)**
4. 勾选 `repo` 权限，设置过期时间（建议90天）
5. 点 **Generate token**，**复制生成的 token**（只显示一次！）
6. 再次执行 `git push` 时，用户名输入你的 GitHub 用户名，密码输入这个 token

### Q: npm install 报错怎么办？

最常见的原因是 Node.js 版本太旧。确保你安装的是 v18 以上的 LTS 版本：

```bash
node --version
# 应该显示 v18.x.x 或 v20.x.x
```

### Q: 网站部署了但是空白页面怎么办？

90%的情况是 `vite.config.js` 中的 `base` 路径和你的仓库名不一致。确认：
- 仓库名是 `my-dashboard` → base 应该是 `'/my-dashboard/'`
- 注意前后都有斜杠 `/`

### Q: 我只想在本地看，不想公开怎么办？

执行 `npm run dev` 就可以在本地浏览器预览，不需要上传到 GitHub。
